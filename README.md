# Lottie WhatsApp Sticker Builder

## Patched: WhatsApp has patched the vulnerability that allowed custom images in WAS; now changing the image has no effect.

Build `.was` sticker files from an image and send them through WhatsApp using Lottie templates with embedded base64 assets.

Portuguese docs:

- [README_ptbr.md](./README_ptbr.md)
- [TUTORIAL_ptbr.md](./TUTORIAL_ptbr.md)

## Requirements

- Node.js 18+
- Official Node.js download: https://nodejs.org/en/download

## Installation

```bash
npm install
```

You do not need to install `zip` on the system. The `.was` file is generated with a Node dependency, so the project behaves the same on Windows, Linux, and macOS.

## Beginner Tutorial

For a step-by-step guide, including Node installation on Windows and Linux, see [TUTORIAL.md](./TUTORIAL.md).

## Language Support

The project supports only two languages:

- `en`
- `pt-BR`

Current behavior:

- the interactive CLI asks which language to use at startup
- the last selected CLI language is saved in `./.lottie-whatsapp.json`
- `build-was.js` and `send-was.js` accept `--lang en|pt-BR`
- `build-was.js` and `send-was.js` also accept `LOTTIE_WHATSAPP_LANG=en` or `LOTTIE_WHATSAPP_LANG=pt-BR`

## Quick Start

```bash
npm run cli
```

For Termux/Android:

```bash
npm run cli:termux
```

The CLI now:

- asks whether you want English or Portuguese
- lists detected images and recent images
- lists detected `.was` files and recent outputs
- validates templates from the registry
- tries to validate or create the WhatsApp session before sending
- adds `.was` automatically to the output name if it is missing
- lets you go back in menus with `Esc`

Local state generated by the CLI:

- WhatsApp session: `./auth_info`
- CLI history/defaults: `./.lottie-whatsapp.json`

Both are ignored by Git and can be removed without affecting the source code.

## Scripts

- `npm run cli`: guided build/send flow
- `npm run cli:termux`: guided build/send flow using `jimp`
- `npm run build:was`: argument-based build flow
- `npm run build:was:termux`: argument-based build flow using `jimp`
- `npm run send:was`: argument-based send flow

## Build From The Command Line

Minimal example:

```bash
npm run build:was -- --image ./your-image.png
```

Full example:

```bash
npm run build:was -- \
  --image ./your-image.png \
  --output ./my-sticker.was \
  --template expand \
  --scale 1.2 \
  --pack-name "My Pack" \
  --publisher "My Name" \
  --emojis "ðŸ˜€,ðŸ”¥" \
  --lang en
```

Options:

- `--image`: input image path
- `--output`: output `.was` path
- `--template`: registered template `id`
- `--scale`: scale applied inside the template asset
- `--pack-name`: pack name shown in WhatsApp
- `--publisher`: pack publisher
- `--pack-id`: pack ID
- `--accessibility-text`: accessibility description
- `--emojis`: comma-separated emojis
- `--base-folder`: override the template base folder
- `--json`: override the JSON path inside the base folder
- `--lang`: `en` or `pt-BR`
- `--no-fit`: disable automatic fit to the asset size
- `--list-templates`: list registered templates

## Send From The Command Line

```bash
npm run send:was -- --to 1234567890 --file ./output.was --lang en
```

Options:

- `--to`: destination number in international format
- `--file`: `.was` file path
- `--auth-dir`: Baileys session folder
- `--lang`: `en` or `pt-BR`
- `--logout`: disconnect after sending

On the first run, the sender prints a QR code in the terminal. After that, it reuses the saved session in `./auth_info`.

## Templates

Available templates live in [templates/registry.json](./templates/registry.json).

Each registry item defines:

- `id`: name used by the CLI and code
- `label`: display name
- `descriptionEn`: English description
- `descriptionPtbr`: Brazilian Portuguese description
- `baseFolder`: template base folder
- `jsonRelativePath`: animation JSON inside the base folder
- `preset`: image-layer preset

Example:

```json
{
  "id": "expand",
  "label": "Expand",
  "descriptionEn": "Wide entrance with a smooth settle.",
  "descriptionPtbr": "Entrada ampla e assentamento suave.",
  "baseFolder": "./src/exemple",
  "jsonRelativePath": "animation/animation_secondary.json",
  "preset": "expand"
}
```

To add your own template:

1. Create the template base folder.
2. Make sure the JSON already contains an embedded `data:image/...` asset.
3. Add a new entry to `templates/registry.json`.
4. Use the new `id` in the CLI or code.

## Use In Code

```js
const path = require("path");
const { buildLottieSticker } = require("./src/index");

async function main() {
  const output = await buildLottieSticker({
    imagePath: path.resolve("/path/to/your-image.png"),
    output: path.resolve("/path/to/sticker.was"),
    template: "expand",
    imageScale: 1.2,
    metadata: {
      packName: "My Pack",
      publisher: "My Name"
    }
  });

  console.log(output);
}

main();
```

Main `buildLottieSticker` parameters:

- `imagePath`: input image path
- `buffer`: alternative to `imagePath`
- `mime`: explicit MIME when using `buffer`
- `output`: output `.was` path
- `template`: registered template `id`
- `baseFolder`: override the template base folder
- `jsonRelativePath`: override the template JSON path
- `fitToAsset`: resize the image to the asset size
- `imageScale`: extra scale applied after fitting
- `metadata`: optional pack metadata overrides

## Notes

- the project does not create animations from scratch
- the Lottie template must already contain an embedded base64 image asset for replacement
- the final look depends on the chosen template
